它能做什么

围绕"看 HTML 文档"这件事做到底，每一个细节都为减少打扰服务。

📂 嵌套目录树 可拖拽

按一级项目目录自动归类。可手动新建子分组、拖拽文件/分组重排。物理文件位置不动，只在虚拟层重组。

🔴 未读红点 基于 mtime

AI 改过哪个 HTML，文件名后立即出红点。点开预览即清除——你能精准锁定"还没看过"的内容。

🔔 桌面通知 SSE + chokidar

HTML 文件新增/修改时弹系统通知。⚙ 设置 里一键开关，浏览器原生 Notification API。

✎ 备注名 不改源文件

给文件起一个你能记住的别名（"数据报告 v3"），原文件名不变。备注存在 Atlas 自己的 store 里。

📂 在访达 / 资源管理器中显示 跨平台

每个文件的 📂 按钮，macOS 用 open -R、Windows 用 explorer /select、Linux 用 xdg-open。

🖱 拖拽调宽 + 收侧栏 ⌘B

侧边栏宽度可拖拽调整，⌘B / Ctrl+B 一键收起。iframe 不会因为侧栏切换而抖动。

🔎 搜索 + 仅未读 / 聚焦

搜索框匹配文件名、备注、相对路径。/ 一键聚焦，「仅未读」开关快速过滤。

📁 多扫描根 设置面板

在 ⚙ 设置 里增删扫描根目录，立即生效。配置存在用户目录 ~/.atlas/，不会动你的项目文件。

三步上手

不需要开发环境。一条 npx 命令搞定首次运行。

1

启动

不需要安装。直接 npx：

npx atlas-dashboard

或者全局安装后用 atlas 命令：

npm i -g atlas-dashboard
atlas
2

首次配置

第一次启动会问几个问题（全部都有合理默认值，回车跳过即可）：

👋 欢迎使用 Atlas

? 要扫描哪些目录的 HTML？(逗号分隔)
  默认: ~/Documents
? 监听端口？
  默认: 4321
? 要忽略的目录名？
  默认: node_modules,.git,...

✓ 已写入 ~/.atlas/config.json
3

访问

浏览器打开：

http://localhost:4321

建议加到书签栏。然后让你的 AI 不要再 open HTML——把规则加到 AI 的全局配置（如 ~/.claude/CLAUDE.md）即可。

命令

前台启动用于调试，后台守护进程用于日常常驻。

命令 作用
atlas前台启动（首次会引导配置；Ctrl+C 退出）
atlas start后台启动，立即返回，写 PID 与日志
atlas stop停止后台服务
atlas restart重启后台服务
atlas status查看运行状态（pid / url / uptime / 扫描根）
atlas logtail -f 日志
atlas init重新进入交互引导（仅配置，不启动）
atlas --port <n>临时换端口（不写回配置）
atlas --root <path>临时加扫描根（可重复，不写回配置）
atlas --config打印配置 / 日志 / PID 文件位置
atlas --help查看完整帮助

让 AI 不再每次打开 HTML

Atlas 解决了"散乱 + Tab 爆炸"，但自动 open xxx.html 是 AI 端的行为。 把这条规则加到 AI 工具的全局配置（如 ~/.claude/CLAUDE.md）： 

生成或更新 HTML 文档后，不要执行 open / xdg-open 打开浏览器。
本机运行着 Atlas dashboard（http://localhost:4321），
它会自动通过 mtime 显示未读红点。
